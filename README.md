1. Некорректное объявление вектора для налогов
  Было: vector<int> Taxes(Z); - использовалась переменная Z вместо N
  Стало: vector<int> Taxes(N); - правильный размер вектора

  Переменная N хранит количество артефактов (первое число в файле)
  Переменная Z хранит минимальный вес, который нужно превысить (второе число в файле)
  Налоговое бремя (di) указано для каждого из N артефактов
  Если использовать Z как размер вектора, то:
    При Z > N - вектор будет больше, чем нужно (появятся "пустые" элементы)
    При Z < N - вектор будет меньше, часть данных не поместится
    При чтении файла произойдет выход за границы массива или потеря данных
  Только размер N гарантирует, что для каждого артефакта будет место для хранения его налога

2. Ошибка в условии цикла при поиске минимального налога
  Было: for (int w = Z + 1; w < MaxWeight; ++w) - не включал последний элемент
  Стало: for (int w = Z + 1; w <= MaxWeight; ++w) - корректный диапазон

  Нам нужно найти вес, который строго больше Z (Z + 1 и выше)
  Максимально возможный вес - это MaxWeight (сумма весов всех артефактов)
  При использовании < MaxWeight последний элемент с индексом MaxWeight не проверяется
  Если оптимальное решение использует все артефакты (вес = MaxWeight), оно будет пропущено
  Исправление <= гарантирует проверку всех возможных весов от Z+1 до MaxWeight включительно


3. Неправильный порядок вывода результатов
  Было: Сначала выводились индексы, потом вес и ценность
  Стало: Сначала выводится суммарный вес и ценность, затем индексы в правильном порядке

  В задании явно указан порядок: "Вывести так же суммарный вес и общую ценность результата"
  Сначала должны выводиться числовые характеристики решения
  Индексы артефактов выводятся последними, как дополнительная информация
  Это улучшает читаемость и соответствует требованиям
  

4. Проблема с обратным порядком индексов
  Было: Индексы выводились в порядке добавления (обратном)
  Стало: Индексы выводятся в порядке возрастания через reverse итерацию

  Алгоритм восстановления решения проходит по артефактам с конца (N-1 до 0)
  Поэтому первый найденный артефакт имеет наибольший индекс
  Без реверса индексы выводятся в порядке убывания (например: "5 3 1")
  После исправления индексы выводятся в порядке возрастания (например: "1 3 5")
  Это более логично для пользователя и соответствует нумерации артефактов
